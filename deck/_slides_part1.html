PROS & CONS

bash 改在 VM 跑：得到什麼、失去什麼

＋優點 · 自包含任務
炸不到 host
  • 破壞性指令（rm -rf、誤刪）都關在 VM
  • 轉檔、資料清理、編譯、跑測試、裝套件都很順
  • 可放心讓 agent 直接執行它生成的程式碼
－缺點 · 要碰 host
runtime 只能用 VM 的
  • kubectl / docker 對 host 叢集、daemon 跑不了
  • python 只能用 VM 內的，不是 host 配好的環境
  • 不能直接對 host 下 git、操作 host 服務
  • 對外連線受 egress 白名單限制
